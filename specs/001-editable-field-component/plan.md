## Summary

Build two custom elements for the `@substrate-system/editable-field`
package:

1. **`<pencil-button>`** (`src/edit-btn.ts`) — a standalone icon button
   that renders the Tabler pencil SVG with a `.visually-hidden`
   accessible label and fires standard `click` events.

2. **`<editable-field>`** (`src/index.ts`) — wraps a disabled
   `<input>` and a `<pencil-button>`. When the button is activated the
   input is enabled and focused.

Both elements extend `@substrate-system/web-component`, register via
`define()`, and live in the light DOM with no Shadow DOM.

## Implementation Notes

### PencilButton (`src/edit-btn.ts`)

```ts
connectedCallback () {
    if (this.querySelector('button')) return  // idempotency guard
    this.innerHTML = `<button type="button">
        <svg ...><!-- penci.svg markup --></svg>
        <span class="visually-hidden">Edit</span>
    </button>`
}
```

### EditableField (`src/index.ts`)

```ts
connectedCallback () {
    if (this.querySelector('input')) return   // idempotency guard
    const name = this.getAttribute('name')
    const value = this.getAttribute('value') ?? ''
    this.innerHTML = `
        <input
            id="${name}"
            name="${name}"
            value="${value}"
            disabled
            aria-disabled="true"
        />
        <pencil-button></pencil-button>
    `
    this.querySelector('pencil-button')
        ?.addEventListener('click', () => this._enableEdit())
}

_enableEdit () {
    const input = this.querySelector('input')
    if (!input) return
    input.removeAttribute('disabled')
    input.removeAttribute('aria-disabled')
    input.focus()
}
```

### attribute forwarding

`name` and `value` are in `observedAttributes`. Their
`handleChange_name` / `handleChange_value` handlers update the
internal input's corresponding attribute directly.
